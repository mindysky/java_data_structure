## 七大设计原则

#### 1. 单一职责原则

- 对类来说，一个类应该只负责一项职责

- 提高类的可读性，可维护性
- 降低变更引起的风险
- 通常情况下，我们应当遵守单一职责原则，只有逻辑足够简单，才可以在代码级违反单一职责原则；
- 只有类中方法数量足够少，可以在方法级别保持单一原则



#### 2. 接口隔离原则 interface segragation principle

- 客户端不应该依赖它不需要的接口，即一个类对另一个类的依赖应该建立在最小接口上
